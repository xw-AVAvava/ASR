# Submission Checklist

Before submission, create one folder containing:

- final video, at least 10 minutes, presented in English
- project source code in this repository
- `docs/final_report.md`
- `docs/architecture.md`
- `docs/contribution.md`, filled with real names and contribution percentages
- `docs/github_url.md` or another markdown/txt file containing the GitHub repository URL
- selected generated outputs from our own audio, for example `outputs/real_asr_base_cluster` or the latest final run
- optional slides

Do not include teacher-provided reference projects as our own project output.

Then:

1. Zip the folder.
2. Upload the zip file to Google Drive.
3. Send only the Google Drive sharing link to the professor by private WeChat or email.

Important:

- Each team has 6 students.
- Only one member needs to submit for the team.
- The professor may check GitHub commit history.
- Each member should make meaningful commits.
